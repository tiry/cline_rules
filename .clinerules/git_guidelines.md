## Git Commit Guidelines

* **Wait for explicit user confirmation** before committing.
* During ongoing development, it's fine to **amend previous commits** to maintain a clean history.
* Each commit message should:

  * Be **5–10 lines** long
  * Reference the corresponding spec file

## Git init Guideline

When the project is created and not yet associated to a Git Repository you should propose to create a Git Repository and associate it with the project.
